@Anotaciones!

Las anotaciones son principalmente utilizadas para reemplazar los arhivos de configuración XML y pueden cambiar el comportamiento del Framework drásticamente.
Vamos a ver como implementar nuestros primeros Beans y una configuración inicialsimple.
En primer lugar debemos crear una clase e importar: 

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

Esto permitirá que podamos empezar a usar las anotaciones @Configuration y @Bean

Las clases de @Configuration son clases que contienen definiciones bean (métodos anotados con @Bean) o son auto-configuradas para identificar definiciones bean en la aplicación; anotándolas con @ComponentScanning.

Ahora si ya podemos empezar a crear la clase equivalente al app-context.xml: 

@Configuration
public class ConfiguracionAnotaciones {
    // equivalente a <bean id="proveedor" class=".."/>
    @Bean
    public MsgProveedor provider() {
        return new MsgProveedorImpl();
    }

    // equivalente a <bean id="renderizador" class=".."/>
    @Bean
    public MsgRender renderer() {
        MsgRender renderizador = new MsgRenderImpl();
        renderizador.setMsgProveedor(provider());
        return renderizador;
    }
}

Finalmente, el método main() debe utilizar la implementación de ApplicationContext que determina como serán interpretadas las definiciones @Bean de las clases de @Configuration
Esta clase es AnnotationConfigApplicationContext (si asi se llama). 

public static void main(String[] args) {
    ApplicationContext ctx = new AnnotationConfigApplicationContext
                                (ConfiguracionAnotaciones.class);
    MsgRender mr = ctx.getBean("renderer",MsgRender.class);
    mr.render();
}
